# 1. Введение

## Цели
В этом документе представлена спецификая требований к программному обеспечению для системы обучения персонала(СОП). Он предназначен для всех сотрудников компании. В нём описаны область применения системы, функциональные и нефункциональные требования к программному обеспечению, ограничения при проектировании и системные интерфейсы.

## Границы применения
Система обучения персонала предназначена для развития квалификации сотрудников с помощью знаний других сотрудников (внутренний обмен опытом). Она обеспечивает основу для:

Формирования и регистрации потребностей в обучении (заявки).

Планирования и бронирования внутренних обучающих мероприятий (лекций, семинаров).

Учета и оценки проведенного обучения (отзывы и оценки усвоения).

Система не является платформой для размещения готовых курсов или внешней системой управления обучением (LMS).
## Термины и аббревиатуры
СОП	Система Обучения Персонала.

SRS	Software Requirements Specification (Спецификация требований к программному обеспечению).

Тимлид	Руководитель команды, инициирующий заявку на обучение для своих сотрудников (Заказчик обучения).

Преподаватель	Сотрудник, откликающийся на заявку и проводящий обучение (Исполнитель).

Обучаемый	Сотрудник, посещающий занятие и оставляющий отзыв.

LMS Внешняя система управления обучением.

# 2. Общее описание

## Описание изделия
Система СОП является веб-приложением, работающим в корпоративной сети. Система не зависит от внешних систем на первом этапе, но спроектирована с учетом возможности будущей интеграции с корпоративными календарями и HR-системами.
- **Интерфейсы системы**: Веб-интерфейс, доступный через браузер. REST API для потенциальной будущей интеграции с корпоративными системами.
- **Интерфейсы пользователя**: Адаптивный веб-интерфейс с отдельными представлениями для Тимлидов, Преподавателей и Обучаемых. Включает дашборды, формы создания заявок, каталог мероприятий и систему отзывов.
- **Интерфейсы аппаратных средств ЭВМ**: Не требуются. Система является веб-приложением, развертываемым на серверах компании. Пользователи взаимодействуют с системой с помощью стандартных клиентских устройств (ПК, ноутбуки).
- **Интерфейсы программного обеспечения**: Интеграция с корпоративной службой каталогов (Active Directory/LDAP) для аутентификации. Серверная часть использует СУБД (например, PostgreSQL) для хранения данных.
- **Интерфейсы коммуникаций**: Все взаимодействия с системой должны осуществляться по защищенному протоколу HTTPS.
- **Ограничения памяти**: Система должна быть рассчитана на хранение данных о всех сотрудниках компании, истории заявок, мероприятий и отзывов за несколько лет. Требования к дисковому пространству должны быть определены исходя из расчетного количества пользователей и активности.
- **Действия**: Система обрабатывает следующие основные операции: создание/редактирование заявок на обучение, отклик на заявки с предложением времени, бронирование мероприятия, запись сотрудников на мероприятие, расчет минимального количества участников, сбор и хранение отзывов и оценок.

## Функции изделия
1. Управление заявками на обучение (создание, просмотр, редактирование)

2. Система откликов преподавателей с предложением временных слотов

3. Бронирование мероприятий и планирование расписания

4. Управление квотами участников:

	Установка максимального количества участников преподавателем

	Автоматический контроль минимального количества участников системой

5. Регистрация сотрудников на мероприятия

6. Система отзывов и оценок:

	Отзывы обучаемых о качестве занятий

	Оценки преподавателей об усвоении материала

7. Назначение конкретных сотрудников на мероприятия тимлидами

## Характеристики пользователей
| Роль         | Описание                                     | Уровень подготовки       |
|--------------|----------------------------------------------|--------------------------|
| Тимлид       | Руководитель команды, инициатор обучения     | Уверенный пользователь ПК|
| Преподователь| Внутренний эксперт, проводящий обучение      | Уверенный пользователь ПК|
| Обучаемый    | Сотрудник, посещающий занятия                | Базовый пользователь ПК  |
| Администратор| Технический специалист, настраивающий систему| Эксперт                  |

## Ограничения
1. Регистрация пользователей только через корпоративную систему (Active Directory)

2. Отсутствие глобального ограничения на максимальное количество участников (ограничение устанавливает преподаватель)

3. Минимальное количество участников устанавливается системой и не может быть изменено пользователями

4. Отсутствие интеграции с внешними календарями на первом этапе

5. Система не поддерживает самостоятельную регистрацию пользователей

## Предположения и зависимости
1. Все пользователи имеют стабильный доступ к корпоративной сети и веб-браузеру

2. Система зависит от доступности корпоративной службы каталогов для аутентификации

3. Пользователи добросовестно используют систему (своевременно вносят данные, не злоупотребляют отменами)

4. Функциональность email-уведомлений не является обязательной для базовой версии

5. Система развертывается на стандартных серверах без требования специального оборудования
